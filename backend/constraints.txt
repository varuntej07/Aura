# Why this exists: the Docker build installs into a completely clean
# container, so pip has to resolve pyproject.toml's dependency graph from
# scratch every time. Nearly every dependency here is a bare `>=X` floor with
# no upper bound, and livekit-agents pulls in several tightly-pinned
# transitive deps (click, aiohttp, opentelemetry-*) that fight with those
# loose floors. Pip's backtracking resolver tries to search the ENTIRE
# version history of packages like click/multidict/opentelemetry/
# google-cloud-tasks to find a compatible set and gives up with
# "resolution-too-deep" before finding one — even though a compatible set
# provably exists (this file IS that set). Passing `-c constraints.txt` to
# pip install skips the search entirely: it doesn't add new requirements, it
# just tells pip the exact version to pick the moment a package IS needed.
#

# bound locally, run the full test suite + both import checks, then
